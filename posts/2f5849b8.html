Lecture 1

Lecture Organization

  • In this lecture we will first get introduced to TCP/IP protocol stack- Its history and evolution.
  • Then the lecture will focus on IP protocol: The packet header of IP in some detail.
  • This lecture will then review IP addressing/address resolution concepts that were covered in COMP2250.
  • Finally, very briefly, it will discuss ICMP a companion protocol for IP.

TCP/IP

Set of communications protocols that implement the protocol stack on which the Internet and most commercial networks run.

Internet Protocol (IP)

Network layer protocol for routing data across Internet.

IP Versions

  1. Version 4: IP networks using IP version 4.
    1. Address length 32 bits.
    2. Address space divided into classes.
    3. Not sufficient to handle projected number of users.
    4. Classes limit the available addresses.
  2. IP Version 6: Also known as IPng (ng: New generation).
    1. Uses 128 bit addresses.
    2. Simplified/flexible packet format.
    3. Support for security features.
    4. Handles congestion and route discovery better than version 4.

OSI Model

TCP/IP versus OSI

Primary IP Function

  1. Accept data from upper transport layer (TCP/UDP)
  2. Create a datagram
  3. Route datagram through internetwork.
  4. Deliver datagram to recipient transport layer and finally the intended application.
  5. Need to examine datagram before the above bullet points.

IP Datagram (IP version 4 Header)

IPv4 Header

Header Length

  1. This is a 4-bit field.
  2. Two cases:
    1. HLEN=5 (actual length= 5*4=20 bytes)-Standard (minimum) size
    2. HLEN=15 (actual length= 15*4=60 bytes)
  3. Presence or absence of Options field.
  4. Exercise: In an IP packet, the value of HLEN is 1000 in binary. How many bytes of options are being carried in the packet?

Type of Service

  1. Current Interpretation: Codepoint (6 bits)DSCP + 2 bits (flow control and are called explicit congestion notification (ECN) bits).
  2. The first 3 bits of DSCP defines what is known as “Per Hop Behaviour”.

Per Hop Behaviour (PHB)

  1. Rule that governs how packets are handled within a Differentiated Services (diffserv) network is called the PerHop Behaviour (PHB).

  2. The PHB assigns a priority to a traffic class, which indicates the precedence flows of that class in relation to other traffic classes.

  3. What is a Traffic Class?

  4. Traffic classification is an automated process which categorises computer network traffic according to various parameters (for example, based on port number or protocol) into a number of traffic classes.

    ​ Each resulting traffic class can be treated differently in order to differentiate the service implied for the user (data generator/ consumer).

  5. In theory, a network could have up to 64 (i.e. 26) different traffic classes using different DSCPs.

  6. In practice, however, most networks use the following commonly defined per-hop behaviors:

    1. Default PHB — which is typically best-effort traffic

    2. Expedited Forwarding (EF) PHB — dedicated to lowloss, low-latency traffic

    3. Assured Forwarding (AF) PHB — gives assurance of delivery under prescribed conditions

    4. Class Selector PHBs — which maintain backward compatibility with the IP Precedence field.

Other Fields

  1. Total length
  2. Time to Live (add)
  3. Protocol
  4. Header Checksum
  5. Source IP address
  6. Destination IP address

IP Fragmentation

  1. Technique used by layered protocols.
  2. Higher level protocol packet placed in data portion of lower-level protocol packet.
  3. Example: IP datagram is Placed within a Data Link frame.
  4. Fragmentation: Division of packet into smaller units to accommodate a protocol’s MTU.
  5. Each fragment has its own header.
  6. Fragment can be further fragmented.
  7. Datagram fragmented at source or any other router in the path.
  8. Reassembly done only at destination.
  9. During fragmentation, required parts of header copied.
  10. Three fields specifically modified.
    1. Checksum recalculated. Fragmented header of IP packet.
    2. Fragment ID (Identification) –16 bit: Identifies all fragments that belong together.
      1. ID with source IP for unique identification.
      2. IP maintains a counter to label this field.
      3. ID copied in all fragments.
    3. Flag-3bit: Indicates other fragments follow.
      1. 1st bit reserved.
      2. 2nd bit =DF bit. (1, 0)
      3. 3rd bit = MF bit. (1, 0)
  11. Fragment offset-13bit: Tells the receiving host how to reassemble the packet.

Fragmentation Exercise

A packet has arrived with an M bit value of 1. Is this the first fragment, the last fragment, or a middle fragment? Do we know if the packet was fragmented?

IP Fragmentation - Key Question

  • Assume an intermediate router has volunteered to do packet de-fragmentation on behalf of end hosts.
  • Would that be a good idea?!
  • If yes, why? If not, why?

Fragmenting a Fragment

  1. If previously fragmented first and middle fragment is fragmented, MF of all fragments set to 1.

  2. If previously fragmented last fragment is fragmented, all the fragments except the last one will have MF set to 1.

    ​ Only 1 fragment has MF set to 0 regardless of the number of times fragmentation is performed.

  3. Offset always calculated relative to the original datagram.

  4. Receiver does not see multiple layers of fragment.

IP Options

  1. IP header: Fixed part + Variable Part
  2. Fixed Part: 20 bytes (all fields except the ‘options’ field)
  3. Variable part: 40 bytes (includes Options)
  4. Not a mandatory field: Usually for network testing and debugging.

Categories of Option

Record Route Option

Strict Source Route Option

Loose Source Route

  • Includes a sequence of IP addresses (like SSR).
  • Allows multiple network hops between successive addresses in the list.
  • List produced resembles to that of Record Route.

Timestamp Option

Addressing at Network Layer: IP Addressing

  1. Internet a combination of different physical networks.

  2. Packets travel different physical networks to destination.

  3. A global identification system necessary for hosts (and routers).

  4. Solution: Unique identifier is IP address in Internet.

  5. IP address:

    ​ Unique

    ​ Universal.

  6. Location not names.

IP Address Format

Address Classes

Multihoming

  1. Internet address defines node’s connection to its network.

  2. Multihomed: A device connected to more than one network.

    1. Different address for each network.
    2. Address can be of different classes.
    3. A unique address associated with each interface.

  3. Can you think of such a multihomed device??

    Router

Alternate Definitions:

  • Availability of two or more connectivity providers to offer fault tolerance and traffic engineering capabilities Or simply,
  • a host is considered multihomed if it has multiple IP addresses

Special Addresses

  • Some parts of address space (A,B,C) are used for special addresses.
  • Network address: A,B.C, address with hostid set to zero.
    • Defines the network itself.
  • Direct Broadcast Address
  • Limited Broadcast Address
  • This host on this network
  • Specific host on this network
  • Loopback Address
  • Unicast-Multicast-Broadcast
  • Private Addresses

Subnet Mask

What is a subnet mask?

  • Distinguishes network id from host id.
  • Specifies if destination host is local or remote.

Default subnet mask

  • Used on networks not divided into subnets.
  • What is a subnet??
  • All hosts on TCP/IP network need a subnet mask.
  • Its use depends on address class.
  • All bits corresponding to network id are set to 1.
  • All bits corresponding to host id are set to 0.

Default subnet mask

  • Class A: 255.0.0.0
  • Class B: 255.255.0.0
  • Class C: 255.255.255.0

Determining local/remote hosts

  • ANDing: Internal process to determine whether a packet is local or remote.
  • Host’s IP address and remote address ANDed with default subnet mask.
  • If both match, they belong to same network.

Subnet: RFC 950

  • Division of a single class A, B, or C network into smaller pieces.
  • Each piece: A physical network in TCP/IP environment.
  • Uses IP address derived from single network ID.
  • Result: Single network (Single Netid) divided into smaller subnets.
    • Each subnet has different network ID.

Subnetting

Benefits:

  • Mix different technologies.
  • Overcome limitations of current technologies.
  • Reduce network congestion.
  • Subnetworks appear as single network to the rest of Internet.

Variable Length Subnetting

  • Scheme to divide a subnet further into smaller subnets.
  • Subnetted network uses more than one subnet mask.
  • Mask can be of variable length: Variable Length Subnet Mask (VLSM)

VLSM Benefits:

  • Efficient use of Organization’s assigned IP address space.
  • Route aggregation.

  • An organization has been assigned the network number 140.25.0.0/16 and it plans to deploy VLSM. Figure in the previous slide provides a graphic display of the VLSM design for the organization.
  • Specify the sixteen subnets of 140.25.0.0/16
  • Identify the broadcast address for Subnet #3 (140.25.96.0)
  • Specify the 16 subnets of Subnet #14 (140.25.192.0/19)
  • Identify the broadcast address for Subnet #14-3

Address Resolution

  • Mapping: To map logical address to physical address and vice versa.

  • Mapping:

    ​ Static versus Dynamic

Address Resolution Protocol (ARP)

  • Successful mapping of IP (Logical) address to a hardware (Physical) address.
  • Address resolution is function of ARP.
  • ARP uses local broadcast to obtain a hardware address.
  • Address mappings are stored in cache for future reference.

ARP: Resolving Remote Addresses

  • ARP request initiated
    • IP address identified as remote.
  • Determine IP address of default gateway.
  • Check cache for hardware address of gateway
  • If not found, generate ARP request for gateway’s hardware address
  • Get gateway response.
  • Send data packet to gateway.
  • At gateway, determine address local or remote.
  • If local, broadcast ARP request, obtain hardware address of the destination.
  • Else, determine default gateway and repeat the process.

Internet Control Message Protocol (ICMP): RFC 792

  • A network layer protocol.
    • Encapsulation.
  • Functions:
    • Error reporting
    • Host/management queries.
  • No error correction feature.

Lecture 2

Lecture Organization

  • This lecture will first introduce you to the concept of IP routing. It will define routing, highlight the difference between forwarding and routing, highlight routing principles and explain the route installation process.
  • It will then look into the various routing methods used in routing table design.
  • It will then examine static and dynamic routing techniques followed by a discussion on routing algorithms and protocols
  • It will finally conclude with a brief discussion on IP Forwarding.

IP Routing

  • Routing: Process of choosing a path over which to send packets.
  • Routing occurs at TCP/IP host when it sends IP packets and at an intermediate IP router.
  • IP module central to Internet technology.
  • Essence of IP is its routing table.
  • IP uses this in-memory table to make all decisions about routing an IP packet.

Routing versus Forwarding

  • Forwarding: Taking a packet, looking at its DA, consulting the table, sending the packet accordingly on the right interface.
  • Routing: Process by which forwarding tables are built.
    • Complex distribution algorithms.
  • Forwarding T able versus Routing T able: Separate data structures for the two preferred.
    • Optimizing lookup, optimizing topology calculation.
    • Implementation

Routing Principles

  • Decision has to be made as to where the packet is to be forwarded. (router/host)
  • IP layer consults routing table in memory.
  • By default, a router can send packets only to networks to which it has configured interface.

Communication Steps

  • IP first determines if destination host is local or on a remote network. (Applying subnet mask).
  • If remote, check routing table for a route to remote host or remote network.
  • If no explicit route is found, IP uses default
    route.
  • At the default router,
    • Consult routing table, if a path is not found,
      use default gateway address
    • Process repeated, until packet finally delivered
      to destination host.
  • If route not found, error message sent to
    source.
    • an ICMP “host unreachable” or “network
      unreachable” is sent back to original sender.

Route Installation Task

  • Main Routing table (RIB) has several sources of information available to reach a given destination
    • Interfaces
    • Static configuration
    • Dynamic updates

Administrative Distance
An administrative distance is the metric used by routers to choose the best path when there are two or more routes to the same destination from two different routing protocols.

Routing Methods

Methods necessary to make the size of routing table more manageable.

Methods

  1. Route Specific
  2. Next-Hop
  3. Network Specific
  4. Host Specific
  5. Default
  6. Source routing (Route Pinning)

Routing Table (Forwarding Table)

Routing table

  • IP performs the following steps when it searches its routing table:
    • Search for Direct delivery
    • Search for a matching host address.
    • Search for a matching network address.
    • Search for a default entry.
  • Default entry normally specified in routing table as network entry, with a network ID of 0.
  • A matching host address is always used before a matching network address.

Routing Strategies

  • Routing can either be static or dynamic.
  • Dynamic strategies are deployed using Routing Protocols

Static Routing

  • Require that routing tables are built and updated manually.

Dynamic Routing

  • Routers update dynamically.
  • A function of routing protocols such as RIP and OSPF .
  • Routing Protocol based.
  • Note: Routing table needed in both the schemes

Types of Dynamic Routing Schemes

  • Distance Vector Routing (Discussed under RIP)
  • Link State Routing (Discussed under OSPF)
  • Hierarchical Routing (used in Link State)
  • Broadcast Routing

Distance Vector Routing: Router

  • Gather knowledge about the whole network.
  • Share the knowledge only with neighbours.
  • Information shared at regular intervals.
  • Cost: In terms of hops. Example: RIP

Link State Routing: Router

  • Gather knowledge about neighbourhood.
  • Share the knowledge with everyone in the network.
  • Information shared only when there is a change.
  • Cost: On variety of factors. Example OSPF.

Hybridized Routing

  • Hybridized Routing: Combination of useful properties of LINK and distance vector Routing.
  • Link State Disadvantages: Logistics and Expense
  • Initially relies on flooding
  • Does sophisticated Calculations: CPU Time and memory.

IP Forwarding

Routing Table Search: Classful Addressing

  • Routing Table is organized as a list.
  • To make searching more efficient, the routing table can be divided into three tables (sometimes called buckets), one for each class.
  • When the packet arrives, the router applies the default mask (which is inherent in the address itself) to find the corresponding bucket (A, B, or C).
  • The router then searches the corresponding bucket instead of the whole table.

  • What do you understand by the term ‘default routing’? Is default routing necessarily a static route?
  • What do you understand by the term ‘static null route? Why would you use such a route?
